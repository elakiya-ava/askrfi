## 1. Problem Statement

Avalere Health consultants manually fill RFIs (Requests for Information) from pharma/biotech clients. Each RFI is an Excel workbook with 50вҖ“200+ questions covering company info, compliance, data security, ESG, staffing, and more.

**Pain points:**
- 60вҖ“70% of questions are recurring across clients (same question, same answer)
- Consultants spend 4вҖ“8 hours per RFI on boilerplate answers they've written before
- No central system connects past answers to new questions
- Base info exists on SharePoint but isn't structured for quick retrieval
- Risk of inconsistent answers across RFIs for the same client

---

## 2. Goals

### v1 Goals (2-week sprint)
1. **Auto-fill boilerplate questions** вҖ” static company facts that never change
2. **Match and adapt past answers** вҖ” find similar questions from the 20 filled RFIs and reuse/adapt answers
3. **Flag gaps** вҖ” clearly mark questions the system couldn't confidently answer
4. **Preserve Excel formatting** вҖ” output is a new copy of the original Excel, not a stripped-down version

### Non-Goals (v1)
- Web UI / dashboard
- SharePoint API integration (blocked by InfoSec)
- .docx / .pptx format support
- Real-time collaboration
- Auto-submission to clients
- Fine-tuning any model

> **Note:** The Parser is a **Python module** (not an LLM agent). It handles Excel I/O deterministically using openpyxl. The pipeline has **2 LLM agents** (Matcher+Filler, Reviewer) and **2 Python modules** (Parser, Writer). There is no separate Classifier agent вҖ” category inference is handled within the Matcher+Filler agent using sheet name hints and Azure AI Search's built-in filtering.

### Component Descriptions

#### Module: Parser (Python вҖ” no LLM)
- **Input:** Path to new RFI Excel file (.xlsx or .xlsm)
- **Output:** Structured list of questions with cell references
- **Logic:**
  - Open workbook with openpyxl (read-only mode for .xlsm to preserve macros)
  - Iterate all sheets; skip instruction-only sheets
  - Detect question/answer column structure (heuristic: look for "Question"/"Response"/"Answer" headers, or longest text column)
  - Handle merged cells by unmerging and attributing to first cell
  - Output: `List[{sheet_name, row, col, question_text, answer_cell_ref, existing_answer}]`

#### Agent 1: Matcher + Filler
- **Input:** List of extracted questions from Parser
- **Output:** Draft answers with confidence scores and traceable citations
- **Logic:**
  - **Category inference:** Use sheet name hints from Parser as category signal; no separate classification step. Azure AI Search handles semantic matching across all categories.
  - **Match (vector search):** For each question, query Azure AI Search Question Index вҶ’ get top-3 similar past Q&A pairs with similarity scores
  - **Match (client):** Client name may be used as a soft signal but should NOT be over-indexed on вҖ” answers from other clients are equally valid if semantically relevant
  - **Boilerplate (similarity > 0.90):** Use past answer verbatim or near-verbatim
  - **Strong match (0.70вҖ“0.90):** Send to Claude: "Adapt this past answer to the new question"
  - **Weak match (0.50вҖ“0.70):** Send to Claude: "Using these references, draft an answer"
  - **No match (< 0.50):** Search Knowledge Index for base info вҶ’ if found, Claude synthesizes; if not found вҶ’ flag as empty
  - **Confidence scoring:** ALL confidence scores (including boilerplate) are generated by the LLM вҖ” no hardcoded values. The LLM assesses confidence based on source quality, relevance, and completeness.
  - **Citations:** Every answer MUST include traceable citations back to source documents (past RFI filename + question number, or base info document + section). Citations must be structured and machine-readable for auditability.
  - Output: `List[{question, answer, confidence, citations: [{source_file, section, question_id}]}]`

#### Agent 2: Reviewer + Writer
- **Input:** Draft answers with confidence scores
- **Output:** Filled Excel file + summary report
- **Logic:**
  - **Validate:** Check for contradictions (e.g., two answers give different employee counts)
  - **Write Excel:**
    - Copy original file to `{filename}_FILLED.xlsx`
    - Write answers into identified answer cells
    - Add "Confidence" column (or use next empty column)
    - Color-code answer cells: green (вүҘ0.80), yellow (0.50вҖ“0.79), red (<0.50)
    - Preserve all original formatting, dropdowns, merged cells
  - **Summary report:** Generate `{filename}_SUMMARY.md` listing fill rate, flagged questions, sources used

## 5. Knowledge Base Strategy

See [knowledge-retrieval-strategy.md](./knowledge-retrieval-strategy.md) for full research.

**Summary:** Hybrid Vector RAG with Azure AI Search dual-index (Question Index + Knowledge Index), Azure-hosted embeddings, semantic ranker.

### Indexing Pipeline (one-time setup + incremental updates)
1. Parse all 20 filled RFIs вҶ’ extract Q&A pairs вҶ’ embed questions вҶ’ store in Azure AI Search Question Index
2. Parse base info docs (PDF/HTML вҶ’ text) вҶ’ chunk by section вҶ’ embed вҶ’ store in Azure AI Search Knowledge Index
3. Extract metadata: category (from sheet name hint), source file, question ID for citation traceability
4. Client name stored as metadata but NOT used as a primary retrieval signal

## 6. Input / Output Specifications

### Input
- **New RFI file:** `.xlsx` or `.xlsm` file path
- **CLI command:** `rfi-agent fill <path-to-rfi.xlsx> [--client <name>] [--output-dir <path>]`

### Output
- **Filled Excel:** `{original_name}_FILLED.xlsx` вҖ” new copy with answers + confidence column + color-coding
- **Summary report:** `{original_name}_SUMMARY.md` вҖ” fill rate, flagged items, sources
- **Categorized workbook (optional):** If the input RFI isn't organized by category, produce a `{original_name}_CATEGORIZED.xlsx` with questions grouped into category sheets

### CLI Interface
```bash
# Fill a new RFI
rfi-agent fill "path/to/new_rfi.xlsx"

# Fill with explicit client name (improves matching)
rfi-agent fill "path/to/new_rfi.xlsx" --client "Pfizer"

# Re-index the knowledge base (after adding new past RFIs)
rfi-agent index

# Show stats about the knowledge base
rfi-agent stats
```

## 7. Edge Cases & Mitigations

| Edge Case | Mitigation |
|---|---|
| Merged cells in Excel | Unmerge, attribute content to top-left cell |
| Questions spanning multiple rows | Detect by: row has text but no answer column content, next row continues |
| Dropdown / validation cells | Preserve data validation in output; if answer must match a dropdown option, constrain generation |
| .xlsm macros | Open in read-only mode; copy file byte-for-byte, only modify answer cells via openpyxl |
| Very long questions (>512 tokens) | Truncate for embedding, but send full text to Claude |
| Ambiguous category | Assign primary + secondary category; search both in vector store |
| Same question worded differently | Handled by semantic similarity (embeddings catch paraphrases) |
| Empty / instruction-only sheets | Skip sheets with <3 question-like rows |
| Non-English content | Flag for human review; don't auto-fill |
| Password-protected Excel | Fail gracefully with clear error message |
| Contradictory past answers | Use most recent answer; flag discrepancy in summary |
