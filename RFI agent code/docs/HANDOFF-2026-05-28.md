## 1. Critical Bug Fixes in `agents.py`

### Bug A: `category_hint` vs `category` Field Mismatch

**Problem:** `excel_parser.py` sets `category_hint` on each question, but `match_and_fill()` read `q.get("category", "Uncategorized")`. Since no `category` field exists, it always resolved to `"Uncategorized"`, which matched nothing in the local KB.

**Fix (line ~348):**
```python
category = q.get("category_hint") or q.get("category", "Uncategorized")
```

### Bug B: No Fallback When Category Has No KB Match

**Problem:** When the local JSON KB had no entries for a category, the LLM received an empty context string → blank answers.

**Fix (lines ~367-375):** When `similar_qas` is empty and `base_info_snippets` is empty, fall back to ALL base_info files concatenated. This guarantees the LLM always has organizational context.

### Bug C: Whitespace in `question_text`

**Fix:** Added `.strip()` before the empty-question check to handle whitespace-only cells.

### LLM Prompt Updates
- Removed instruction to prefix answers with `[NEEDS REVIEW]`
- Added: "Leave the answer blank if you cannot answer"
- Added: "Set confidence to 0 for questions you cannot answer"

Both sync `match_and_fill` and async `_fill_one` were fixed identically.
